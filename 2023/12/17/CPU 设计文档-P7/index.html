定义模块

以自下而上的方式搭建

{add, sub, and, or, slt, sltu, lui
addi, andi, ori
lb, lh, lw, sb, sh, sw
mult, multu, div, divu, mfhi, mflo, mthi, mtlo
beq, bne, jal, jr }。

阶段 简称 功能概述
取指阶段（Fetch） F 从指令存储器中读取指令
译码阶段（Decode） D 从寄存器文件中读取源操作数并对指令译码以便得到控制信号
执行阶段（Execute） E 使用 ALU 执行计算
存储阶段（Memory） M 读或写数据存储器
写回阶段（Writeback） W 将结果写回到寄存器文件

  • Tuse：ID 级，其后的某个功能部件再经过多少时钟周期就要使用寄存器值。
  • Tnew：再经过多少周期就能够产生要写入寄存器的结果。

思考题

一、请查阅相关资料，说明鼠标和键盘的输入信号是如何被 CPU 知晓的？

通常是接入USB端口，经过主板上的南桥芯片组处理，经PCIe将信号送到CPU。

二、请思考为什么我们的 CPU 处理中断异常必须是已经指定好的地址？如果你的 CPU 支持用户自定义入口地址，即处理中断异常的程序由用户提供，其还能提供我们所希望的功能吗？如果可以，请说明这样可能会出现什么问题？否则举例说明。（假设用户提供的中断处理程序合法）

中断是一种异步事件，可能在任何时刻发生，而处理中断需要 CPU 能够快速找到相应的中断处理程序的位置。使用预先指定的地址，例如中断向量表中的地址，可以有效地实现这一点。允许用户自定义中断处理程序可能引入安全漏洞、影响系统性能，并导致不一致的系统行为。

三、为何与外设通信需要 Bridge？

外设通信需要 Bridge，因为它可以转换不同总线标准、速度、协议和处理地址映射，以确保各组件协同工作。

四、请阅读官方提供的定时器源代码，阐述两种中断模式的异同，并分别针对每一种模式绘制状态移图。

两种中断都会导致pc跳转到0x4180，模式0倒计时归零后停止，直到enable再次置一，用于产生定时中断。模式1倒计时归零后重新计数，用于产生周期性脉冲。

当前状态 下一状态 触发条件
IDLE LOAD|IDLE 控制信号触发 (CTRL[0])
LOAD CNT 预设值加载完成
CNT INT 计数器减到1
INT IDLE 控制信号清零 (CTRL[0])

五、倘若中断信号流入的时候，在检测宏观 PC 的一级如果是一条空泡（你的 CPU 该级所有信息均为空）指令，此时会发生什么问题？在此例基础上请思考：在 P7 中，清空流水线产生的空泡指令应该保留原指令的哪些信息？

在检测到中断信号时，如果在宏观PC的一级是一条空泡指令，可能导致中断响应延迟。在 P7 中，清空流水线产生的空泡指令应该至少保留原指令的 PC 信息与延迟槽信号，以确保正确存入EPC，以便返回到中断前的执行状态。

六、为什么 jalr 指令为什么不能写成 jalr $31, $31？

jalr 指令将 GPR[rs]写入PC，同时将当PC+4保存在GPR[rd]中。CPU在宏观上是单周期CPU，若rs和rd相同，会导致数据冲突。
